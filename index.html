Software Size Estimation from Conceptual Model

This simple tool helps you estimate potential the size of a software product from its conceptual design model (CDM)

What is the rationale?

Any software application is essentially action on data, movement and its persistence

An application database is the most efficient way the day is stored, the relationship represents how the data could be acted upon and the number of attributes represents a quantitative representation of its magniture. A relationship among these factors succinctly represent all that transformations and its extent that could happen on the represented abstraction. A regression model built on the interelationships form the crux of this tool.

How to use this tool?

There are three parameters based on which an estimate is prepared:
Number of Classes
The number of classes or entities defined in a Conceptual Design Model (CDM).
Number of Relationships

The relationships are drawn as lines connecting entities in a CDM. Each unidirectional relationship is counted once. You will get just enough details from my paper on the subject here . So, if there are bi-directional relationships, they would need to be counted twice.

Average number of attributes

This number is the average count of all attributes defined in the CDM: total number of attributes in the conceptual model divided by the number of classes.

Accuracy ?

The tool uses a linear regression model for the estimate. The LOC estimation model has an MMRE and Pred(0.25) values of 0.14 and 0.81, which are comaprable to the Function Point method. The accuracy of the estimate is within 75% 81 percent of the time.
